## What This Is

This is a structured SQL portfolio built around one idea — SQL isn't just about passing a query, it's about solving a real problem.

Every case in this portfolio starts with a problem from LeetCode or similar platforms, but instead of just solving it for the sake of it, each one is reframed as a real business scenario. The goal is to document not just what the query does, but why it's written that way, what business decision it enables, and where it breaks down at scale.

This is not a collection of solutions. It is a record of applied thinking.

---

## What Each Case Includes

- **Business scenario** — a realistic context mapped to the actual problem
- **Table structure** — schema with data types explained
- **Sample data** — realistic data that reflects the scenario
- **SQL solution** — clean, formatted, with comments where needed
- **Approach breakdown** — why this query, why not something else
- **Alternate approach** — what else could work and why it wasn't chosen
- **Performance note** — where this scales well and where it doesn't
- **Business impact** — what decision or action this query enables
- **LinkedIn post** — link to the published post documenting the case

## Why I Built This

Most SQL portfolios are just a list of solved problems with no context. A recruiter or hiring manager looking at that has no way to understand how you think — only that you can pass a query.

This portfolio is built differently. Every case connects the technical solution to a business outcome. The goal is to demonstrate not just SQL ability, but analytical thinking — knowing what to query, why to query it that way, and what the result actually means for a business.

---

## Who This Is For

This portfolio is relevant if you are looking for someone who can:

- Translate a business requirement into a SQL query
- Write clean, readable, production-aware SQL
- Think critically about query performance and edge cases
- Communicate technical work in plain business language
